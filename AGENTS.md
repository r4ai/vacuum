# vacuum

ビルド成果物や依存関係キャッシュを再帰的に削除する CLI ツール。
使い方は README.md を参照。

t-wadaのTDDで実装する。

## 文章

簡潔で分かりやすい文章を書く。

- 1文が短く簡潔
- 論理が明快
- 意味を一意に解釈できる（読み違いが発生しない）

## テスト

**古典派テスト（振る舞いの検証）** を採用する。
実装の詳細ではなく、外部から観察可能な振る舞いをテストする。
これにより、リファクタリングで壊れない堅牢なテストを実現する。

以下の2階層でテストする。

- 単体テスト：関数単位でロジックを検証する。副作用が大きく難しいものは無理にテストしなくてよい。
- 結合テスト：実際にCLIを呼び出し、全体を通して正常に動作するかを検証する。これがメイン。原則モックなどは使わず、実際に動かして正常に動くことをテストする。
